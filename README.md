# 竞争对手店铺上新监控脚本

这是一个使用 Python 编写的自动化脚本，能够定期（例如每 6 小时）抓取指定页面的商品列表，与历史数据比对，并发送邮件通知新增商品。

## 准备工作

### 1. 安装依赖

确保你已经在这个环境中安装了 Python，并且需要安装 `requests` 和 `beautifulsoup4` 这两个库。
打开终端（比如 PowerShell 或 cmd），进入脚本所在的 `d:\JetBrain\Project\store-monitor\` 目录，运行：

```bash
pip install requests beautifulsoup4
```

### 2. 获取邮箱授权码
由于脚本需要通过 SMTP 自动发送邮件，你不能直接使用邮箱的登录密码，需要生成一个“授权码”。
- **QQ邮箱**：进入设置 -> 账号 -> 开启 POP3/SMTP 服务 -> 点击“生成授权码”。
- **网易(163)邮箱**：进入设置 -> POP3/SMTP/IMAP -> 开启 SMTP 服务 -> 新增授权码。
将这段授权码复制下来，稍后填入 `monitor.py`。

### 3. 配置脚本
打开 `monitor.py` 文件，修改文件最上方的 `<⚙️ 配置区域>`：

1. **`TARGET_URL`**: 目标监控页面的完整 HTTP 地址。
2. **`SENDER_EMAIL`** 和 **`RECEIVER_EMAIL`**: 发件邮箱和收件邮箱（发给自己填同一个即可）。
3. **`SENDER_PASSWORD`**: 第 2 步获取到的邮箱授权码。
4. **`SMTP_SERVER`**: 你的邮箱提供商的 SMTP 服务器。例如 QQ 填 `smtp.qq.com`，163 填 `smtp.163.com`。

### 4. 调试网页抓取规则 (🔴 核心步骤)
每个网站的代码都不一样，所以这一步需要根据你要监控的网页手动配置一下。
在 `monitor.py` 的 `fetch_products` 函数中找到以下代码：

```python
# ⚠️ 注意：这里的选择器需要你根据目标网站的源码(F12)进行修改！
items = soup.find_all('div', class_='product-item') 
```

**如何找这个 Class？**
1. 用浏览器浏览器打开你要监控的网址。
2. 按快捷键 `F12` 打开开发者工具。
3. 点击开发者工具左上角的“箭头”图标，然后在网页上去点击一个具体的商品。
4. 在弹出的 HTML 代码里，看看包裹这个商品的层叫什么 Class 名字或者结构，替换到上面的代码里。

> **如果自己改不好，你可以把目标 URL 发给我，我帮你查源码并改写这段选择器的逻辑代码。**

## 运行监控

在终端里运行：

```bash
python monitor.py
```

- 脚本长亮运行，**第一次运行** 会获取当前页面商品存入本地（不会发邮件，因为没有对比参照）。
- **之后每隔 6 小时**，脚本会再次去访问网页并对比，挑出新出现的商品给你发邮件通知。
